# AISQL：企业级智能数据协同与增强检索平台

**AISQL** 是一个易用、高性能的分布式智能数据助手。它深度整合了 **Text-to-SQL** 与 **RAG (Retrieval-Augmented Generation)** 技术，旨在消除自然语言与结构化数据库、非结构化知识库之间的壁垒，为企业提供秒级响应的“对话即分析”能力。

-----

## 🎯 项目旨在解决的问题

在企业日常运营中，数据获取通常面临以下痛点：

1.  **维度缺失：** 业务统计指标（SQL）与制度文档（PDF/Markdown）相互隔离，无法在一个入口获取完整信息。
2.  **查询门槛高：** 非技术人员难以编写复杂的 SQL 进行多表关联，导致数据分析需求大量堆积在技术团队。
3.  **响应延迟：** 传统的报表提取流程繁琐，无法满足实时决策的需求。
4.  **冗余录入：** 知识库在更新维护时，容易产生重复的向量分块，影响检索精度并增加存储成本。

-----

## 🚀 要实现的目标

  * **超融合存储架构：** 基于 **PostgreSQL + pgvector** 实现业务数据与知识向量的同库存储，简化系统复杂度。
  * **智能 Text-to-SQL：** 支持自然语言自动转 SQL、执行查询，并具备基于 **LangGraph** 的报错自动纠错（Self-Healing）能力。
  * **精准 RAG 检索：** 针对企业文档进行语义切片，利用 **MD5 哈希指纹** 实现数据入库的去重校验。
  * **安全脱敏引擎：** 内置 PII（个人敏感信息）识别模块，在 Prompt 阶段自动屏蔽敏感数据，符合企业合规标准。
  * **多维结果展示：** 自动识别查询结果类型，动态生成可视化图表。

-----

## 🛠️ 技术栈

| 模块 | 技术实现 |
| :--- | :--- |
| **LLM 编排** | LangChain / LangGraph (Multi-Agent 工作流) |
| **数据库** | **PostgreSQL (pgvector)** |
| **后端架构** | FastAPI (支持异步任务处理) |
| **向量检索** | HNSW / IVFFLAT 索引优化 |
| **任务队列** | Celery + Redis (处理大规模 Embedding 计算) |
| **数据同步** | Kafka / Flink (感知数据变动并准实时更新向量) |
| **前端交互** | Vue.js |

## 📅 阶段计划 (Roadmap)

### 第一阶段：核心能力构建

  - [ ] 设计 PostgreSQL 业务 Schema 与向量字段融合方案。
  - [ ] 开发基础 Text-to-SQL 链条，支持多表关联识别。
  - [ ] 实现基于内容哈希（MD5）的向量入库去重逻辑。

### 第二阶段：生产级特性增强

  - [ ] 接入 **LangGraph** 实现 Agent 自动纠错工作流。
  - [ ] 优化 `RecursiveCharacterTextSplitter` 策略，支持长文本的语义完整性。
  - [ ] 增加数据库连接池管理与查询性能分析。

### 第三阶段：企业级赋能

  - [ ] 开发通用数据脱敏插件（PII Masking）。
  - [ ] 支持通过 Excel/PDF 批量上传构建私有知识库。
  - [ ] 增加对话上下文记忆（Session Management）功能。
