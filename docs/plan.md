采用 **Monorepo (frontend/backend)** 架构，优先攻克“用户体系”与“动态数据源配置”这两个基石。

---

## 🏗️ 项目架构全景图
**aisql** 将作为一个全栈 AI 应用运行，前端负责交互与可视化，后端负责 AI 编排与数据库通信。

---

## 📅 阶段一：基础架构与环境搭建
**目标：** 建立开发规范，实现“一键启动”的开发环境。

### 1.1 仓库初始化 (Monorepo)
- [ ] **目录结构：** 创建 `apps/frontend` 和 `apps/backend`。
- [ ] **容器化：** 编写 `docker-compose.yml`，集成：
    * **PostgreSQL 16+** (安装 `pgvector` 扩展，如 pgvector/pgvector:pg18-trixie)。
    * **Redis 7** (用于 Celery Broker 和缓存)。

### 1.2 后端基础 (Backend Foundation)
- [ ] **技术选型：** FastAPI + SQLAlchemy。
- [ ] **用户系统：** 实现 JWT 认证，支持用户登录/注册。
- [ ] **安全层：** 引入 `cryptography` 库，用于加密存储用户的业务库密码。

### 1.3 前端基础
- [ ] **技术选型：** Vue.js + Tailwind CSS。
- [ ] **用户系统：** 实现 JWT 认证，支持用户登录/注册。

---

## 📅 阶段二：数据源管理与元数据感知
**目标：** 让系统能够“连接”并“理解”外部数据库。

### 2.1 动态数据源配置
- [ ] **API 开发：** 实现业务库连接的 CRUD（增删改查）。
- [ ] **连接测试：** 编写校验逻辑，支持在保存前实时测试 MySQL/PostgreSQL 连通性。
- [ ] **Schema 提取：** 编写元数据反省（Introspection）逻辑，自动抓取表名、字段名及注释（Comment），存入系统元数据表。

### 2.2 前端配置面板 (Vue 3)
- [ ] **UI 实现：** 使用 Tailwind CSS 编写简洁的数据库连接表单。
- [ ] **交互反馈：** 实现连接测试的 Loading 状态和成功/失败的 Toast 提示。
- [ ] **Schema 预览：** 连接成功后，展示该数据库的表结构树状图。

---

## 📅 阶段三：Text-to-SQL 核心引擎开发
**目标：** 实现从自然语言到 SQL 的转化与执行。

### 3.1 Prompt 工程与 Chain 编排
- [ ] **Context 注入：** 将阶段二提取的 Schema 信息动态注入 Prompt。
- [ ] **SQL 执行器：** 实现安全的只读 SQL 执行环境，支持多表 Join 识别。
- [ ] **结果处理：** 后端自动判断结果集（是单值、列表还是趋势数据），为前端返回对应的 JSON 结构。

### 3.2 纠错 Agent (LangGraph)
- [ ] **自愈逻辑：** 如果 SQL 执行报错，将 Error 信息反哺给 LLM，自动触发 1-2 次修正尝试。

---

## 📅 阶段四：RAG 增强与“超融合”存储
**目标：** 引入非结构化知识，实现“数据+政策”的混合检索。

### 4.1 知识库构建 (pgvector)
- [ ] **文档切片：** 针对企业文档（PDF/Markdown）优化切片策略。
- [ ] **去重机制：** **核心功能**。入库前计算文本 MD5 哈希，利用 `ON CONFLICT DO NOTHING` 避免冗余。
- [ ] **索引优化：** 针对海量向量数据，手动创建 **HNSW 索引**。

### 4.2 混合查询路由
- [ ] **意图识别：** 判断用户提问是查“数值统计”还是“政策解读”，自动路由至 SQL 引擎或 RAG 引擎。

---

## 📅 阶段五：数据可视化与上线调优
**目标：** 提升用户体验，完成项目交付。

- [ ] **可视化渲染：** 前端集成 **ECharts**，根据后端返回的数据类型自动渲染柱状图或折线图。
- [ ] **数据脱敏 (PII)：** 在返回前端前，对敏感信息（如手机号、姓名）进行掩码处理。
- [ ] **流式输出：** 实现 WebSocket 或 SSE，让 AI 的回答具备“打字机”效果。
